Github+Hexo搭建个人Blog(一)

准备工作

前言

一直希望拥有完全属于自己的博客，一方面可以做笔记，记录个人成长，同时也可以写写随笔，当作一个小小的精神家园。虽然网上有很多类似简书、CSDN博客园等个人博客网站，但是页面样式个人不是很喜欢，同时还会在你的个人页面插入许多广告，让人反感。于是之前我就租了一个云服务器，搭建了一个基于wordpress的个人博客。但是其费用成本较大，后来到期后就干脆放弃了。后来在网上发现用Github+Hexo可以搭建，方便简洁，重要的是完全免费。好了，言归正传。接下来，让我一步一步教大家搭建一个自己的Blog。

环境搭建

Github账号
用作博客的远程仓库、域名和服务器等。Github官网

Git
用于将本地的Hexo内容提交到Github上。为了方便操作，这里我用了Git Bush。下载地址

gitbush安装完成后，打开gitbush，运行git version命令可以查看其版本。

Node
用于生成静态页面。Node.js官网
除在官网下载安装包安装node外，也可以在gitbush上进行安装。

在gitbush中运行以下下命令行：

  • $ sudo apt-get install -y nodejs
    安装完node.js后，安装npm（node的包管理工具）：
  • $ sudo apt-get install npm

Hexo
Hexo是一款基于Node.js的静态博客框架，让上百个页面在几秒内完成渲染。

先在电脑的磁盘中创建一个文件夹，然后cd到该文件夹下，或则直接在该文件夹下右键,选中git bush here。
使用npm安装Hexo：

  • $ sudo npm install hexo-cli -g

运行hexo -v可以查看当前版本。

初始化命令：hexo init,初始化之后，打开其对应的文件夹可以看到一下文件：

一下对上述文件稍作解释：

  • node_modules：依赖包
  • public:：用处存放生成页面
  • scaffolds：命令生成文章等的模版
  • source：用于命令创建的各种文章
  • themes：主题
  • _config.yml：整个博客的配置
  • db.json：source解析所得到的
  • package.json：项目所需模块及配置信息

以上准备工作基本完成。

搭建github博客

创建仓库

新建一个名为用户名.github.io的仓库，比如说，我的github用户名是wilton_cn,那么就新建一个wilton_cn.github.io的仓库。（必须使用你的用户名，否则无效。）同时，这也是我以后网站的网址，别人就可以通过http://wilton_cn.github.io 来访问我的网站了。（如果你不想另外都买域名的话。）

注意：

  • 注册的邮箱一定要验证，否则不会成功；
  • 仓库的名字必须为username.github,io，其中suername为你的用户名；
  • 仓库创建成功并不会立即生效，需要过一段时间，一般为10-30分钟。

以上一个github仓库便创建成功了，以后你的博客代码以及博文等就都放在这个仓库啦。

绑定域名(不另购域名可跳过)

上面说了，即使你不另外购买域名也可以通过http://xxx.github.io 来访问网站。但是你可能觉得它不够个性，所以你可以另外购买一个域名作为网站的网址。购买域名的平台很多，这里大家就根据个人喜好自行选择吧。不过个人建议还是选一些大的平台比较靠谱，比如阿里腾讯等。
域名都买完成后就需要解析了，一般来说购买了域名都是会送免费解析的。常见的域名配置有两种，CANME和A记录，CNAME填写域名，A记录填写IP。由于不带www只能采用A记录，所以必须线ping一下你的用户名.github.io的IP，然后到你的域名DNS设置页，将A记录指向你ping出来的地址，再将CNAME指向你的用户名.github.io，这样能确保无论是否添加www都可以访问。

如何ping地址：win+R启功命令窗口，输入cmd。然后在命令窗口中输入ping 用户名.github.io,接下来你就会看到对应的IP。

完成以上工作，并没有完成域名绑定，你还要做下面一步工作。在上一节说到的hexo的文件的source文件夹下创建一个CNAME文件，其中内容为你的域名，至于加不加www根据你个人喜好，具体差别如下：

  • 如果你填写的是没有www的，比如wilton.wang，那么无论是访问 http://www.wilton.wang 还是 http://wilton.wang， 都会自动跳转到 http://wilton.wang；
  • 如果你填写的是带www的，比如www.wiolton.wang，那么无论是访问 http://www.wilton.wang 还是 http://wilton.wang， 都会自动跳转到 http://www.wilton.wang；
  • 如果你填写的是带www的，比如abc.wilton.wang，那么只能访问 http://abc.wilton.wang， 否则无效。
    此外，在你绑定了域名之后，你原来的username.github.io并没有失效，而是会是直接跳转到你的新的域名。

配置SSH key

为了能够将你的本地博客内容提交到github仓库，你要拥有github权限才可以，但是如果直接使用用户名和密码不太安全，所以我们使用ssh key来解决本地和服务器的连接的问题。
在git bush中输入$ ssh-keygen -t rsa -C "邮箱地址"
然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

点击保存，完成配置。为了测试配置是否成功，输入以下带命令：
ssh -T git@github.con
如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，如果配置成功，会看到一下内容：

Hi wilton-cn! You’ve successfully authenticated, but GitHub does not provide shell access.

此外，你还需要配置一下内容：

$ git config --global user.name "wilton-cn" # 你的github用户名
$ git config --global user.email "xxx@gmail.com" # 注册github的邮箱

提交本地blog内容到仓库

在提交本地内容到仓库之前我们需要做一些配置，打开hexo文件夹下的 _config.yml 文件，修改配置，建立关联，主要做一下修改（其他的可以根据需求修改）：

在git bush中cd到hexo的文件夹下或者在hexo文件夹下右键git bush here,输入hexo init,完成初始化。
然后运行一下命令行：
$ hexo generate
$ hexo deploy

一些常用命令：

  • hexo new”postName” #新建文章
  • hexo new page”pageName” #新建页面
  • hexo generate #生成静态页面至public目录
  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
  • hexo deploy #将.deploy目录部署到GitHub
  • hexo help # 查看帮助
  • hexo version #查看Hexo的版本

以上，本地的博客内容就被提交到服务器上了。在浏览器中输入域名，你就可以访问你的博客了。
